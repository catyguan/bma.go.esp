游戏世界说明：
- 2D世界，坐标系采用整型数（有符号）表示，由X，Y坐标定位一个点
-- 方向: -x方向为左，+x方向为右， -y方向为下，+y方向为上
-- 常量 1-上,2-下,4-左,8-右
- 基本距离单位 1U = 10000 ，其中10000代表具体含义可由客户端自己定义
- 物体编号，游戏世界中的每个物体，都有唯一的物体编号int
- 物体体积
-- 基地的固定体积 4U*4U
-- 坦克的固定体积 4U*4U
-- 炮弹的固定体积 1U*1U
-- 墙和石头无固定体积
- 坦克炮弹的速度为恒定速度
-- 坦克：每秒4U
-- 炮弹：每秒8U

通用指令说明：
- 纯文本，每个指令以\n结束
- 指令不区分大小写

客户端指令（发送给服务端）：
- 客户端指令的接受控制为1指令/5ms
- JOIN nick teamId
	加入游戏
	nick 为A-Z-0-9，表示客户端的名称，如有重复则该指令响应错误
	teamId（数值为A或B）表示所属队伍A或B的成员，不提供队伍则由服务器自动分配
- CHAT xxxxxx
	游戏前的时候，发送消息给所有等待中的客户端(包括自己)
- STOP
	停止移动
- MOVE
	移动
- TURN dir
	dir 方向, int
- FIRE
	开火
- KILLME
	自爆

服务端指令（发送给客户端）：
- OK
	表示服务端接已收到客户端指令
- INFO xxxxx
	服务器发送给客户端的消息，显示而已，无游戏用途
- ERROR xxxxx
	服务器发送给客户端的错误信息。用于显示在指令接受过程中出现的错误
- BEGIN
	表示游戏开始
- END [teamId]
	表示游戏结束，如果有teamId，表示为获胜队伍；无表示平局
- TEAM teamId tankId tankNick
	组队信息
- EVENT eventId message
	广播游戏世界的事件
	eventId，事件序列号，从1-N的有序递增int
	message为json格式字符串

事件数据格式具体说明：
- ekind： 事件类型，int
	1: 状态改变
	2: 新物体出现
	3: 物体消失
	4: 物体损坏（石头被部分摧毁）
- x，y： 事件关联的坐标，int
- w, h： 事件影响大小,int，可选（物体损坏场景）
- oid： 事件关联的对象编号，int，可选
- okind： 事件关联的对象类型，int，可选
	1: 墙壁，不可穿越，不可摧毁
	2: 石头，不可穿越，可摧毁（部分摧毁）
	3: 基地，不可穿越，可摧毁（游戏结束）
	4: 坦克，不可穿越，可摧毁
	5: 炮弹，不可穿越，可摧毁，准确来说碰撞会发生爆炸
- dir： 事件关联物体的方向，int，可选，当物体方向改变的时候会提供
- speed： 事件关联物体的速度，int，可选，当物体速度改变的时候会提供
